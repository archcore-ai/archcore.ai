## Rule

All user-facing copy on the landing site MUST align with the canonical messaging pack (`archcore_messaging_pack.md` in the repo root).

**Primary phrase (use first):** "Git-native context for AI coding agents"

**Secondary phrase (use second):** "Context engineering for repositories"

**Expanded explanation:** "Archcore helps teams turn scattered repo knowledge into structured context that AI coding agents can find, reuse, and follow."

### Vocabulary preferences

Use: "git-native context", "repo context", "project context", "structured context", "context engineering", "context quality"

Avoid as primary framing: "shared architectural memory", "system context platform", "context engineering platform"

### Copy hierarchy

- **Hero H1:** Primary phrase
- **Meta tags / OG / Twitter:** Primary phrase + messaging pack descriptions
- **Footer tagline:** Primary phrase
- **Section copy:** Use "decisions, rules, plans, and guides" (not "experience")

## Rationale

Consistent positioning across all touchpoints (search results, social shares, page content) strengthens brand recognition. The messaging pack is the single source of truth for how Archcore is described externally.

## Examples

**Good:** "Structure decisions, rules, plans, and guides in your repo so Claude Code, Cursor, Copilot, and other agents work with stronger project context."

**Bad:** "Shared architectural memory for AI coding agents" (outdated primary phrase, now avoided).

## Enforcement

Review all copy changes against the messaging pack before merging. The messaging pack file in the repo root is the canonical reference.
